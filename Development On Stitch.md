This document lists my creation process with Google's **Stitch**, including my prompts, etc.
### Starting Prompt


Create a modern mobile app, featuring a AI intelligent kitchen ingredient organizer, where it features a visual-based artificial intelligence agent, where the screen shows the user's camera, and uses ai to automatically detect ingredients and their containers' capacity level. This will be shown with generated, minimalist ui tracing around the containers and ingredients detected. 



*I want the following screens:*

- a minimalist login & introduction screen, subtle but cool. 
- a startup/home screen, with widgets displaying key takeaways, current stats of ingredient consumption and advice on healthy diet generated based on the current usage.
- a settings page, featuring general switches (light/dark mode, language, etc.) and an option where users can type in requests to more specific report generation directions and styles.
- the consumption dashboard, which is basically a more detailed display of the consumption, shown in a line graph, on the quantity left on each ingredient, and the refills detected. 
- The camera & visual intelligence page, just as mention above in the introduction, this should work like any other visual intelligence apps, using the user's mobile camera, and the AI should make these floating labels and outline tracing of the objects. For this demo creation, just simulate the AI in action.

*Special Requirements*

- Please don't generate any user avatar based on your own idea, make this something by the user's choice.
- I want the ui the be clean and minimalist. Don't use to much color or futuristic decoration. Make it subtle but cool in it's vibrant elements and information. Labels and item outlines in the camera page should be mostly white, creating contrast to the media. The media can be slightly dimmed for better visibility of the UI. Don't add useless animations for 'scanning' or so, but I do want a symbol showing that the AI is in action, like any other mainstream AI agents such as ChatGPT, where they have their own indicators.
- I don't need a big heading of the title of the app throughout all screens. The dashboard should just have a clean title, left-centered, saying 'Dashboard', with an appropriately chosen font, where as the setting just says 'Settings', etc. For the layout, try to make it stylish while not lacking in organization.
- Don't generate an image as a placeholder for the user's mobile camera preview. Please leave it blacked out, and use a small label to indicate that this is where the user's camera preview should be - this is very useful for future development.

*Font Usage*

I want stylish, modern and minimalist fonts. The actual specific fonts are up to you, but consider my preference.
